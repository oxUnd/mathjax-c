mathjax-c 与 MathJax 公式对比

左侧是当前 `mathjax-c` 通过 CLI 直接生成的 PNG，右侧是浏览器中 MathJax 的参考渲染。 这个页面的作用不是证明当前实现已经接近 `MathJax-src`，而是把差距暴露出来，方便你快速决定下一轮修复优先级。

目标：复刻 MathJax-src 的排版行为 当前参考：浏览器 MathJax SVG 素材目录：`comparison_assets/`
怎么看 重点看分式高度、上下标位置、根式覆盖范围、大运算符上下限、以及是否把命令当成普通文本。
当前结论 parser 主链已能识别一部分命令，但 layout 和 script style 仍然与 MathJax 有明显差距。
用途 这个页面可以直接作为后续回归页面，修一个模块就刷新一次，看差异有没有收敛。
